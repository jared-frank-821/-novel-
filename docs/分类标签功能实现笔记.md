# 分类标签功能实现笔记

## 功能目标

点击左侧「分类」按钮后，**中间编辑区**变成分类标签选择界面，左右侧栏保持不变。再次点击「分类」切回编辑器。

```
┌─────────────────────────────────────────────────────────┐
│  顶栏（小说选择器、导出导入）                               │
├──────────┬──────────────────────────────┬───────────────┤
│          │                              │               │
│ 左侧边栏 │      中间区域 (middleView)    │   右侧AI助手   │
│          │                              │               │
│ ·作品信息 │  editor ─→ Tiptap编辑器      │ ·灵感卡片      │
│ ·章节     │  categories ─→ 分类标签      │ ·AI对话       │
│ ·分类 ●  │  (条件渲染,二选一)           │               │
│ ·回收站  │                              │               │
│          │                              │               │
└──────────┴──────────────────────────────────┴───────────────┘
```

## 一、数据层：Zustand Store

### 1.1 数据结构

标签数据存储在每个小说对象上，是一个字符串数组 `selectedTags`：

```typescript
// src/store/useNovelListStore.ts

export interface Novel {
  id: string;
  title: string;
  author?: string;
  description?: string;
  category?: string;
  status?: string;
  coverId?: string;
  createTime: string;
  updateTime: string;
  chapters: Chapter[];
  selectedTags: string[];  // 存放选中标签的数组，如 ['言情', '出轨', '系统', '求生']
}
```

### 1.2 toggleTag Action

```typescript
// 在 store 的 type 定义中添加
toggleTag: (tag: string) => void;

// 在 store 的实现中添加
toggleTag: (tag: string) => {
  const { novels, currentNovelId } = get();
  if (!currentNovelId) return;

  const updatedNovels = novels.map(novel => {
    if (novel.id !== currentNovelId) return novel;

    // 核心逻辑：如果标签已存在则移除，不存在则添加
    const selectedTags = novel.selectedTags.includes(tag)
      ? novel.selectedTags.filter(t => t !== tag)      // 取消选中
      : [...novel.selectedTags, tag];                 // 追加选中

    return { ...novel, selectedTags };
  });

  set({ novels: updatedNovels });
},
```

**toggleTag 的设计哲学：**
- **选中 → 点击 = 取消选中**：从数组中 filter 掉
- **未选中 → 点击 = 选中**：spread 追加到数组末尾
- 一行代码搞定两种状态切换，不需要 if/else 分支

### 1.3 持久化

Store 使用 `persist` 中间件，数据自动保存到 `localStorage`，键名为 `'novelList'`。`selectedTags` 会随 `novels` 数组一起持久化，无需额外处理。

## 二、UI 层：CategoriesContent 组件

### 2.1 为什么提取为独立组件？

有两种使用场景：
1. **独立页面**：`/try/categories` 路由，有完整的页面布局（背景色、居中容器）
2. **嵌入模式**：作为 `page.tsx` 中间区域的子组件，紧凑布局

所以抽取为共享组件，通过 `compact` prop 区分两种布局。

### 2.3 分类定义（静态数据）

```typescript
type Category = {
  id: string;       // 唯一标识
  label: string;    // 显示名称
  totalCount: number; // 该分类最多选几个标签
  tags: string[];   // 该分类下的所有标签选项

const CATEGORIES: Category[] = [
  {
    id: 'theme',
    label: '题材',
    totalCount: 1,   // 只能选1个题材
    tags: ['言情', '现实情感', '悬疑', '科幻', '武侠', ...],
  },
  {
    id: 'plot',
    label: '情节',
    totalCount: 3,   // 最多选3个情节标签
    tags: ['出轨', '系统', '权谋', '婚姻', '重生', ...],
  },
  {
    id: 'mood',
    label: '情绪',
    totalCount: 3,
    tags: ['求生', '纯爱', 'HE', 'BE', '甜宠', '虐恋', ...],
  },
  {
    id: 'timespace',
    label: '时空',
    totalCount: 1,
    tags: ['古代', '现代', '未来', '架空', '民国'],
  },

```

### 2.4 反向索引：标签 → 分类

`TAG_TO_CAT` 用于判断某个标签属于哪个分类：

```typescript
const TAG_TO_CAT: Record<string, string> = CATEGORIES.reduce((acc, cat) => {
  cat.tags.forEach((t) => { acc[t] = cat.id; });
  return acc;
}, {} as Record<string, string>);

// 效果：{ '言情': 'theme', '出轨': 'plot', '求生': 'mood', ... }
```

**为什么需要这个索引？**  
`selectedTags` 是扁平数组 `['言情', '出轨', '系统', '求生']`，只知道标签名，不知道它属于哪个分类。而判断某个分类是否达到上限时，需要统计该分类下已选了几个标签。

### 2.5 从 Store 读取数据

```typescript
// 获取当前选中的小说 ID
const currentNovelId = useNovelListStore((s) => s.currentNovelId);

// 获取 toggleTag action
const toggleTag = useNovelListStore((s) => s.toggleTag);

// 从 novels 数组中找出当前小说，返回其 selectedTags
const selectedTags = useNovelListStore((s) => {
  if (!s.currentNovelId) return [];
  return s.novels.find((n) => n.id === s.currentNovelId)?.selectedTags ?? [];
});
```

**Zustand 的 selector 模式：**
```typescript
// 推荐：精确选择需要的字段，避免不必要的重渲染
const currentNovelId = useNovelListStore((s) => s.currentNovelId);

// 不推荐：整个 store，任何字段变化都会触发重渲染
// const store = useNovelListStore();
```

### 2.6 将扁平 tags 转换为分类结构

store 里的 `selectedTags` 是扁平的 `string[]`，但 UI 需要按分类分组展示：

```typescript
const selectedMap: Record<string, string[]> = CATEGORIES.reduce((acc, cat) => {
  acc[cat.id] = selectedTags.filter((t) => TAG_TO_CAT[t] === cat.id);
  return acc;
}, {} as Record<string, string[]>);

// 效果：{ theme: ['言情'], plot: ['出轨', '系统'], mood: ['求生'], timespace: [] }
```

默认值兜底逻辑（用户首次使用没有任何标签时显示预设）：

```typescript
const INIT_MAP: Record<string, string[]> = {
  theme: [],
  plot: [],
  mood: [],
  timespace: [],
};

// 渲染时：
const selected = selectedMap[cat.id] ?? INIT_MAP[cat.id] ?? [];
```

### 2.7 selectTag 核心逻辑

点击标签时执行，包含**上限控制**：

```typescript
function selectTag(tag: string, catId: string) {
  const catDef = CATEGORIES.find((c) => c.id === catId);
  const limit = catDef?.totalCount ?? 1;  // 该分类最多选几个

  // 统计当前分类已选了多少个标签
  const sameCatTags = selectedTags.filter((t) => TAG_TO_CAT[t] === catId);

  // 如果是"选中"操作（tag 不在已选中列表）且已达上限，则忽略
  if (!sameCatTags.includes(tag) && sameCatTags.length >= limit) return;

  toggleTag(tag);  // 调用 store，更新 Zustand
}
```

**上限控制的三个条件：**
1. `!sameCatTags.includes(tag)` — 当前操作是"选中"，不是"取消"
2. `sameCatTags.length >= limit` — 已经选到上限了
3. 两个条件同时满足 → return，忽略本次点击

### 2.8 标签样式

根据是否选中动态切换样式：

```tsx
<span
  onClick={() => selectTag(tag, cat.id)}
  className={`
    inline-flex items-center rounded px-2.5 py-1 text-sm cursor-pointer select-none
    ${selected.includes(tag)
      ? 'border border-emerald-500 bg-emerald-50 text-emerald-600'  // 选中态
      : 'bg-gray-100 text-gray-700 hover:bg-gray-200'               // 默认态
    }
  `}
>
  {tag}
</span>
```

## 六、关键设计决策

### 为什么不直接用 URL 路由切换？

**方案 A（路由跳转）：** 用户点击 → `router.push('/try/categories')` → 整个页面切换，左侧栏重新挂载

**方案 B（状态切换）：** 用户点击 → `setMiddleView('categories')` → 仅中间区域重新渲染，左右侧栏完全不变

本功能选方案 B，核心原因：**用户不需要丢失当前编辑状态**，也不需要等待整个页面重新加载。

### 为什么用 Zustand 而不是 React Context？

- **精确订阅**：组件只订阅自己需要的字段，避免不必要重渲染
- **持久化开箱即用**：`persist` 中间件自动处理 `localStorage` 读写
- **代码简洁**：无需嵌套 `<Provider>`，直接在组件内 `useNovelListStore`

### 紧凑模式如何实现？

通过 CSS 布局分离：

| 模式 | 容器 | 标题大小 | 背景 |
|------|------|----------|------|
| `compact={false}` | 全屏 `min-h-screen bg-gray-100/80 p-6` | `text-2xl` | 独立页面背景 |
| `compact={true}` | 继承父容器 `h-full overflow-auto` | `text-lg` | 父容器背景色 |

共用同一套标签选择逻辑，仅布局容器不同。
